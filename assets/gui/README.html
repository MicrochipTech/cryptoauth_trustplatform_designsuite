Trust Platform Design Suite (1.3.2)

Getting started with Trust Platform GUI

Opening GUI on Windows

  1. Open START → Trust Platfrom V1.3.2 → Trust Platfrom 1.3.2(installation folder name)

Opening GUI on MAC OSX

  1. Open Applications → Trust Platfrom V1.3.2

Menu Description

  1. Design Suite
    1. Setup

      2. - Repository → Helps to select the base folder to operate on by the GUI. Point to an empty folder, to get a new copy cloned from GitHub, or point to existing repository folder, to check its status to interact.

      If GUI finds a Trust Platform Design Suite repo, it starts refering to this while responding to user navigation. If not, it clones the repository from GitHub and set it up for use.

      When the repository is cloned from GitHub, the tool also tries to install the required Python packages. This process can take good amount of time based on network, number of packages and installation time for each module. Wait for setup to be done before closing it. It will display "Python package update - Done" to indicate the completion.

      - MPLAB Path → Trust Plaform notebooks are built with option to program Crypto Auth Trust Platform board with factory program application. Since it needs MPLAB IPE, this option enables user to select the MPLAB version to be used. If this path is not set, user has to program the factory program application manually when the board doesnt have on it.

    2. Clean

      3. - Clicking this option reverts the content of the current repository. It also prompts to keep/remove temporary files like key, certificates, .c, and .h etc..,

    3. Update

      4. - Clicking this option checks GitHub updates for the current repository. It also prompts to update python packages if any available.

    4. GitHub Page

      5. - Opens up GitHub page on web browser.

Buttons

  1. Trust&GO icon

    2. - Opens the html page that describes about Trust&GO device along with usecases available in the selected repository folder.

  2. TrustFLEX icon

    3. - Opens the html page that describes about TrustFLEX device along with usecases available in the selected repository folder.

  3. TrustCUSTOM icon

    4. - Opens the html page that describes about TrustCUSTOM device along with options to configure it. For the complete information on this, its required to have access to TrustCUSOM repository. The access will be provided on request and NDA between User and Microchip.

  4. Getting Started

    5. - Opens the html page that describes about Trust Platform overview including various variants and device part numbers, tools and other links.

  5. Start/Stop Jupyter

    6. - This is the starting point for various usecase notebooks. When Start Jupyter is clicked, notebook kernel should be started to display the Trust Platform content for usecase materails, running notebooks etc.., Once its decided to close the notebook Kernel, click on Stop Jupyter. If the notebook browser is closed in between, user can restart the notebooks by clicking on Stop Jupyter and Start Jupyter again.
